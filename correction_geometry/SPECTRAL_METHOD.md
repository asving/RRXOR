# Rebuilding the machine from behavior: the spectral recipe

(Written up for the Simplex thread. Implementation: `speclib.py` + drivers
`hankel*.py`, `hankel_qa.py`.)

## What is actually reconstructed

The object recovered is an **observable operator model** (OOM / weighted automaton /
PSR — Jaeger; Hsu–Kakade–Zhang), i.e. the *minimal linear-algebraic realization* of
the process: a state vector and one update matrix per symbol. Every m-state HMM is an
OOM of rank ≤ m, and the recovered operators are **similar** (conjugate by an
invertible matrix) to the HMM's observable operators T·diag(O_σ). So states come back
in an arbitrary linear basis — a gauge — and what is canonical is everything
similarity-invariant: the Hankel rank, the spectrum of the operators, and all
predictions. Converting the OOM into a proper HMM (nonnegative matrices) means fixing
the gauge, which needs extra structure; for our tasks we verify against the known
generator instead (rank, spectrum, rollout KL). This is the same "belief state up to
linear transformation" that Simplex probes find in the residual stream — extracted
here from behavior alone.

## The recipe

Inputs: a conditional model f (a checkpoint), a set of real histories {h_i} truncated
at a fixed position (the net is position-dependent, so rows are real contexts, not
abstract strings), a set of test suffixes {s_j}.

1. **Fill the Hankel exactly** — no sampling: H[i,j] = P_f(s_j | h_i) =
   ∏_k f(s_j[k] | h_i · s_j[:k]), computed by teacher-forcing each suffix and
   multiplying the model's own conditionals. Row i is history i's "predictive
   profile" — where it lands in the space of probability-assignments-to-futures.
2. **SVD → belief states.** If the behavior is generated by a rank-m machine, rows
   span an m-dim subspace. Truncate: z(h) = y(h)·V_k (y = row of test probs). z IS the
   belief state, up to gauge.
3. **Operators by shifted-state regression.** The unnormalized Bayes update is linear:
   A_σ z(h) = P(σ|h) · z(h·σ). Both sides are computable from the same conditional
   tables (z(h·σ) needs tests one symbol longer), so each A_σ is a least-squares
   problem over histories. The residual of this regression is a diagnostic: it is ~0
   iff the behavior is *actually* a rank-k machine (mid-training checkpoints fail it —
   they are between machines).
4. **Run it.** p(σ|z) is a fixed linear functional of z (the length-1 test
   coordinates); update z ← A_σ z / p(σ), renormalizing so p(0)+p(1)=1 (without this
   the rollout drifts). A₀+A₁ is the transfer operator; its spectrum = the generator
   transition spectrum (similarity-invariant) — this is where the roots of unity live.

## "Taking the difference" of pre and post

Two inequivalent constructions, both used here:

- **Two machines, compared** (`hankel.py`, `hankel27.py`, `hankel_flow.py`): fit an
  OOM to f_pre and to f_post separately; the drop's effect = what changed (rank,
  new eigenvalues). This needs no notion of "difference" and is the robust version.
- **The correction as its own machine** (`hankel_qa.py`, fit_ratio_wfa): the ratio
  c(s|h) = P_post(s|h)/P_pre(s|h) is multiplicative along paths, so it satisfies the
  same functional equation as a probability (c(σs|h) = c(σ|h)·c(s|hσ)) and the same
  pipeline applies with P(σ|h) replaced by the one-step ratio. Caveat: c is low-rank
  only if 1/f_pre is (reciprocals of WFA functions are not WFAs in general). It works
  when f_pre is a clean local policy — with the idealized bet as f_pre, the correction
  WFA's dominant eigen-triple is {λ₀, λ₀e^{±2πi/3}} — the clock, living inside the
  correction (fit R² 0.95). Against the actual mid-training pre-net it degrades
  (R² 0.83): real mid-training policies are not low-rank.

## Validation battery (numbers from these runs)

- Hankel spectrum vs truth: RRXOR final net matches the generator's rank-5 singular
  values to 3 decimals; matryoshka's true machine is rank exactly 45 and the net
  matches; leaky (fx) machines are rank 5 (the fractal belief SET is a continuum, but
  its linear span — what the Hankel sees — is exactly 5-dim).
- Eigenvalue invariants: RRXOR clock 0.998∠2.0944 (e^{2πi/3} = ∠2.0944); matryoshka
  fills 1 → 3rd → 9th → 27th roots across the drops; leaky clocks at radius ≈ 1−ε
  (true tables: 0.990/0.970/0.800 for ε = .01/.03/.2, exact; nets slightly inside:
  0.983/0.947/0.737 — the net forgets a bit faster than Bayes, the spectral form of
  its CE gap).
- Predictive quality, KL(net‖model) per context position (teacher-forced rollout,
  32 positions): at the first step the reconstructed OOM beats the true ε-machine
  (RRXOR 1.8e-5 vs 1.6e-4; fx 2.1e-4 vs 1.6e-3) because it captures the net's own
  miscalibrations; autoregressive drift crosses over at ~5 steps; 32-step means
  7.5e-4 vs 2.6e-4 (RRXOR) and 6.1e-3 vs 1.4e-3 (fx). Both are far below the
  ~0.5-nat entropy scale.

## Pitfalls (all hit in these analyses)

1. Tests too short → the Hankel sees a low-rank *shadow* and rotation pairs collapse
   onto their real parts (we measured exactly −0.5 = cos 2π/3 before lengthening).
2. Fitting k above the true rank injects null-space garbage that scrambles the
   spectrum.
3. Long-test columns must be per-test normalized (a fixed diagonal map on test space —
   harmless to the realization) or their tiny probabilities are numerically invisible.
4. Rollouts need per-step state renormalization (p(0)+p(1)=1) or drift compounds.
5. The operator-regression R² gates everything: mid-transition checkpoints are not
   rank-k machines and their eigenvalues are interpolation artifacts; read spectra on
   plateaus, or report fit quality alongside (we plot opacity = fit R²).
6. Count root classes by snapping near-unit eigenvalues to the lattice, not by raw
   thresholds.
